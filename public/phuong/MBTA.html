    MBTA Kiosk Redesign Case Study

    Background


    This project is part of my coursework for ENP166, the Computer Interface Design course at Tufts University. The task involved redesigning the interface for the new fare structure ticketing machines.


    Key components required:

    • Include two card options to accommodate different travel modes
    • Get information on how to get from where you are to where you want to go integrating T, bus lines, commuter rail, and ferries
    • Get information about delays/construction/etc.
    • Get information about the Boston area in general
    • Provide information useful to both Boston natives and tourists

    Affordances for:

    • Language selection options.
    • Features designed to assist individuals with low vision or vision impairments.
    • Features tailored for individuals with mobility impairments.

    User Flow
    Kiosk Interface

    With the user personas in mind, I have designed the MBTA Kiosk Interface to fulfill the needs and concerns of users.

    The main screen has . . .

    • 2 main functions
    • INFORMATION contains the latest announcements from the MBTA and a travel guide for both locals and tourists
    • CHARLIECARD allows users to add value to existing passes or purchase a new one
    • 3 accessibility options
      • Adjust text size
      • Change language preference
      • Enable voice assistance
    • HELP or ? button

    If the user chooses to increase the font size, the following changes will be made.

    Users are given six language options to choose from.
    The options were based on the most popular languages in the United States.

    Selecting this button will turn on voice assistance.

    [Voice Assistant] Hello, this is the MBTA! To purchase a CharlieCard, please ...

    How to Purchase a CharlieCard

    The process for purchasing a new CharlieCard is similar to the existing design.
    There are 4 options for a card:

    • Bus/Subway
    • Commuter Rail
    • Savings Passes → reduced fares for weekly or monthy passes
    • Add Custom Amount

    Changes were made to how much value can be added to a CharlieCard for the Subway/Bus.
    Users can chose between:

    • Selecting the specific number of Subway/Bus rides needed → for those who are familiar with the system
    • Inputting a custom amount with OTHER AMOUNT → for those who are new / have a rough estimate of what they need

    The prices for a single Subway or Bus ride is shown on the screen for the user’s reference and convenience.

    At PAYMENT, the kiosk presents the user with:

    • Transit type (student, adult, or senior)
    • Ticket type (ex. savings pass one day/week/month)
    • Amount due
    • Payment options
    • Cancel

    Currently, commuter rail passes are sold on board the trains. In the redesign, users will have the option to purchase their pass in advance. A notable addition is that the Kiosk will prompt the user to input their departure and arrival stations. Subsequently, it will provide an estimated travel time and the corresponding trip cost. This redesign calculates fares based on the distance traveled, ensuring that users are not charged a flat rate.

    • This feature is particularly advantageous for individuals who make short-distance commutes.
    • For long-distance commuters, the price of the fare will flatten past a certain number of stops traveled.
    A Perk for Frequent Commuters

    The current system allows users to purchase unlimited one day/week/month passes. The new addition is the STUDENT SEMESTER PASS. If students forget their T card, they can just scan their ID to purchase a new one.

    Introducing MBTA’s Travel Guide

    One of the primary project requirements was to incorporate a travel guide feature into the interface. This feature is designed to provide users with suggestions for places to explore, with a primary focus on Boston while also encompassing the greater Metro areas. Users can access this function under the INFORMATION section, which also includes the latest announcements from MBTA services, making it a valuable resource for both local residents and tourists.

    New here? Let’s see what Boston has for you!
    Check out the most popular restaurants and entertainment.

    Each list is sorted by distance (based on your current location).

    • The preview includes details such as:
    • Google reviews ratings
    • Closing time
    • T Line nearby
    • Estimated time of travel

    No Map? No Problem!

    If users have a specific destination in mind, the MAPS/NAVIGATION option can provide help with that.
    In NAVIGATION, users can filter out the mode of transportation of their preference.
    After inputting the origin and destination, the results will preview the:

    • Estimated travel time
    • Estimated time of arrival
    • Routes and T line

    Reflection


    In the redesign, I addressed the issues present in the current MBTA Kiosk machine by introducing an interface that enhances user experience and ensures a seamless interaction with the system. As mentioned in the background, I successfully resolved the following concerns:


    • Implemented language selection options
    • Incorporated font size adjustments
    • Added voice assistance functionality
    • Installed a physical HELP button

    I am pleased with the project's outcome; however, if I were to consider making any improvements, I would challenge myself by reimagining the overall design to distinguish it from the existing MBTA design. To achieve this, I would conduct extensive research into transportation systems in other cities and countries, seeking inspiration for innovative concepts and designs.
